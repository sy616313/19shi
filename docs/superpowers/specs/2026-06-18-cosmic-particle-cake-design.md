## 一、设计概述

### 1.1 主题定位

一个以**宇宙星空**为背景、**低多边形粒子蛋糕**为核心的手机横屏生日祝福页面。风格关键词：**梦幻、精致、克制、科技浪漫**。

### 1.2 核心体验

用户打开页面 → 看到宇宙星云中一个由发光粒子构成的两层蛋糕在缓慢旋转 → 蛋糕表面的粒子不时飘出向上消散 → 触摸蛋糕区域粒子散开，松手后粒子优雅回归原位。纯视觉享受，无文字干扰。

### 1.3 与现有需求的关系

本设计针对 `/birthday` 页面的蛋糕部分进行重新设计。将原有的 CSS 手绘粉色蛋糕替换为低多边形粒子蛋糕。保留项目现有的 React 19 + Vite 技术栈。

## 三、蛋糕几何设计

### 3.1 几何体规格

蛋糕由两层**正八边形棱柱**构成，用粒子沿表面分布。

| 属性 | 底层（大） | 顶层（小） |
|------|-----------|-----------|
| 形状 | 正八边形棱柱 | 正八边形棱柱 |
| 半径 | 130px | 95px |
| 高度 | 110px | 85px |
| 粒子数量 | ~800 | ~500 |
| 层间距 | — | 12px（间隙用浮游粒子填充） |

### 3.2 粒子表面分布策略

```
正八边形棱柱展开面：

    ┌─────────────────────────────┐
    │       顶面（八边形）        │  ← 粒子密度：中等
    ├─────────────────────────────┤
    │  侧面1 │ 侧面2 │ ... │ 侧面8│  ← 粒子密度：高（主展示面）
    ├─────────────────────────────┤
    │       底面（八边形）        │  ← 粒子密度：低（不可见居多）
    └─────────────────────────────┘
```

- **面上**：均匀分布，间距 6-8px
- **棱边**：间距稍大（10-12px），避免过于锋利
- **顶面**：中等密度，略带圆弧分布而非严格网格
- **随机偏移**：每个粒子在理想位置上有 ±2px 的随机抖动，打破网格感

### 3.3 粒子视觉规格

| 属性 | 值 |
|------|-----|
| 粒子大小 | 1.5 ~ 3px（面中心偏大，棱边偏小） |
| 形状 | 圆形（`arc()`） |
| 发光效果 | `shadowBlur: 6-10`, `shadowColor` 同粒子色 |
| 透明度 | 0.7 ~ 1.0（面向观察者 > 侧向 > 背向） |

### 3.4 光照模型

模拟一个从**右上方 45°** 入射的平行光源：

```
光源方向：(1, -1, 0.5) 归一化

面法线与光源夹角 → 粒子亮度映射：

  正面（面对观察者）  → opacity 1.0  + 色相偏亮
  右侧面（受光）      → opacity 0.9  + 色相偏亮
  左侧面（背光）      → opacity 0.6  + 色相偏暗
  顶面                → opacity 0.85 + 色相适中
  底面（不可见）       → opacity 0.3  → 不渲染
```

### 3.5 色彩方案

```
蛋糕配色渐变（从亮到暗）：

顶层：
  受光面  ████  #67E8F9 (亮青 Cyan-300)
  中间面  ████  #A78BFA (柔紫 Violet-400)
  背光面  ████  #7C3AED (深紫 Violet-600)

底层：
  受光面  ████  #7DD3FC (天蓝 Sky-300)
  中间面  ████  #C084FC (粉紫 Purple-400)
  背光面  ████  #6D28D9 (暗紫 Violet-700)

层间浮游粒子：
  ████  #E9D5FF (淡紫 Purple-200)  opacity 0.5

飘散粒子：
  ████  #F0ABFC (粉紫 Fuchsia-300) → 渐隐至透明
```

### 3.6 呼吸闪烁

每个粒子有独立的亮度呼吸动画：

- **周期**：2 ~ 4 秒（高斯随机分布，避免同步）
- **幅度**：亮度 ±10%（即 opacity 在 0.63 ~ 0.77 之间波动，对于 base 0.7 的粒子）
- **相位**：随机初始相位，使整体呈现"星空闪烁"效果而非集体闪烁

## 四、粒子动画系统

### 4.1 常态动画（Idle State）

#### 整体旋转

- 绕 Y 轴缓慢自转
- 速度：约 **12 秒 / 周**（30°/秒）
- 实现方式：对所有粒子的 x 坐标做 `cos(θ)` 变换，z 坐标做 `sin(θ)` 变换
- 旋转时自动更新面法线 → 更新粒子亮度（光照随旋转变化）

#### 垂直浮动

- 每个粒子沿 Y 轴微浮动
- 幅度：±3px
- 周期：3 ~ 5 秒（随机）
- 各粒子相位随机 → 整体呈现有机的"呼吸感"

#### 飘散粒子生成

```
生成规则：
  · 频率：每 1.5 ~ 2.5 秒生成一批（2 ~ 5 个）
  · 来源：从蛋糕上表面和侧面随机选取粒子位置
  · 生命周期：
    ┌─ 诞生 ──→ 向上飘散 ──→ 逐渐透明 ──→ 消亡 ─┐
    │  0s        0~3s          2~4s        3~5s    │
    │  size=2.5   速度0.8px/f   alpha衰减   alpha=0 │
    └───────────────────────────────────────────────┘
  · 运动路径：向上 + 正弦横向摆动
    x(t) = x0 + sin(t * 0.8) * 15px
    y(t) = y0 - speed * t
  · 消失点：蛋糕上方 180 ~ 250px 处完全透明
  · 补充：飘散粒子消失时，在原始位置"生长"新粒子
    新粒子从 size=0 渐变到 size=2.5，耗时 0.5s
```

### 4.2 触碰互动（Touch State）

#### 触碰散开（Scatter）

```
触发：手指触摸蛋糕区域（以蛋糕中心为圆心，半径 200px 的圆形区域）

散开动画：
  · 爆炸中心：触摸点坐标
  · 影响半径：触摸点周围 150px
  · 散开方向：从触摸点向外的射线方向 + random(-30°, +30°)
  · 散开速度：3 ~ 8 px/帧（距离触摸点越近越快）
    速度 = base_speed × (1 - distance/150)
  · 粒子状态：亮度 +30%，大小 +0.5px（触碰反馈感）
  · 散开范围：粒子飞出 200px 后停止，悬浮在空中
```

#### 聚合回归（Reassemble）

```
触发：手指抬起后 500ms 延迟

回归动画：
  · 缓动函数：ease-out-back（先快后慢，带微小过冲）
  · 回归路径：不是直线，带弧度
    中间点 = 原始位置 + (当前位置 - 原始位置) × 0.5 + 法线方向 × 30px
  · 回归耗时：800 ~ 1200ms（随机，避免同时到达）
  · 回归完成瞬间：所有粒子同步闪烁一次（opacity → 1.0 → 恢复）
    闪烁耗时 200ms
  · 防抖：散开过程中不接受新触摸
  · 超时保护：持续按住超过 3s，粒子开始缓慢回归（无视触摸状态）
```

### 4.3 动画状态机

```
         touchstart
    ┌──────────────────┐
    │                  ▼
  IDLE ──────────► SCATTER
    ▲                  │
    │    touchend       │
    │   + 500ms delay   │
    │                  ▼
    └──────────── REASSEMBLE
                    │
                  完成聚合
                    │
                    ▼
                   IDLE
```

- IDLE：常态旋转 + 浮动 + 飘散粒子
- SCATTER：散开中，暂停旋转，暂停飘散粒子生成
- REASSEMBLE：回归中，恢复旋转，恢复飘散粒子生成

## 五、技术方案

### 5.1 技术栈

| 层 | 技术 | 备注 |
|----|------|------|
| 框架 | React 19 | 与现有项目一致 |
| 构建 | Vite | 与现有项目一致 |
| 渲染 | HTML5 Canvas 2D | 不引入 WebGL，保持简洁 |
| 动画 | requestAnimationFrame | 自主控制帧率 |
| 背景 | `<img>` + CSS | 静态加载，不参与动画 |

### 5.2 组件结构

```
src/
  components/
    CosmicCake/
      index.jsx            ← 主组件，组装三层
      CakeCanvas.jsx       ← Canvas 渲染引擎（核心）
      useParticleSystem.js ← 粒子系统 Hook（状态 + 物理）
      useTouchInteraction.js ← 触碰交互 Hook
      geometry.js          ← 八边形棱柱生成 + 粒子分布
      config.js            ← 所有可调参数集中管理
```

### 5.3 渲染管线（每帧）

```
requestAnimationFrame 回调：

  1. 更新阶段 (update)
     ├─ 更新旋转角度 θ
     ├─ 更新每个粒子浮动偏移
     ├─ 更新飘散粒子位置 + 生命周期
     ├─ 处理触摸状态（散开/聚合）
     └─ 回收已消亡粒子 + 生成新飘散粒子

  2. 投影阶段 (project)
     ├─ 3D 坐标 → 2D 投影（简单正交投影）
     ├─ 计算面法线 → 更新粒子亮度
     └─ Z-sort（远处粒子先绘制）

  3. 渲染阶段 (render)
     ├─ 清空 Canvas
     ├─ 绘制后层飘散粒子
     ├─ 绘制蛋糕主体粒子
     ├─ 绘制前层飘散粒子
     └─ 绘制层间浮游粒子
```

### 5.4 性能预算

| 指标 | 目标 | 降级策略 |
|------|------|---------|
| 帧率 | 60fps | < 45fps 时减少飘散粒子至 50% |
| 粒子总数 | ~1500 | 降到 ~800 时仍可接受 |
| Canvas 尺寸 | `devicePixelRatio` | 最大限制为 2，超过则降级 |
| 内存 | < 50MB | — |

### 5.5 无障碍与兼容

- 支持 `prefers-reduced-motion`：减少或停止粒子动画，静态展示蛋糕
- 触摸区域最小 44×44px（实际上整个蛋糕区域远大于此）
- 无文字内容，无需 `aria-label`（纯装饰性 Canvas）

## 七、响应式适配

本页面**以手机横屏为主要目标**，但也需基本适配其他场景：

| 场景 | 策略 |
|------|------|
| 手机横屏 (主) | 完整体验，蛋糕按上述尺寸渲染 |
| 手机竖屏 | 蛋糕缩放至 70%，整体下移 10% |
| 平板 | 蛋糕放大至 120% |
| 桌面 | 蛋糕放大至 150%，居中展示 |

缩放通过 Canvas 内部的 `scale` 变量实现，不改变粒子数量。

## 八、动画时间线总览

```
0s      页面加载
        ├─ 背景图加载完成
        └─ 粒子系统初始化（粒子从透明渐入，耗时 1.5s）

1.5s    进入 IDLE 状态
        ├─ 蛋糕缓慢旋转（12s/周）
        ├─ 粒子浮动呼吸
        └─ 飘散粒子周期生成

任意时间  用户触摸
        ├─ 0ms:    触摸点检测，开始散开
        ├─ 0-500ms: 粒子向外飞散
        ├─ 松手后 500ms: 开始聚合
        ├─ 500-1700ms: 粒子回归 + 能量脉冲
        └─ 1700ms: 恢复 IDLE
```

## 九、验收标准

- [ ] 手机横屏打开，宇宙背景图铺满全屏，无拉伸变形
- [ ] 两层低多边形蛋糕居中显示，粒子沿八边形棱柱表面分布
- [ ] 蛋糕缓慢旋转，光照随旋转变化（面法线 → 亮度映射）
- [ ] 粒子有呼吸闪烁效果，各粒子相位不同
- [ ] 飘散粒子从蛋糕表面向上飘出，带正弦摆动，渐隐消失
- [ ] 触摸蛋糕区域，粒子向外散开
- [ ] 松手后粒子优雅回归原位（弧线路径，过冲回弹）
- [ ] 回归瞬间有能量脉冲闪烁
- [ ] 60fps 流畅运行（iPhone 12 及以上）
- [ ] `prefers-reduced-motion` 下动画减弱或停止
